Kafka Banking Pipeline
Data Engineering · Event Streaming
Overview
Architected a high-throughput banking transaction system using Apache Kafka in KRaft mode, simulating real-world financial data pipelines at scale. The system handles atomic transactions, batch processing, and multiple operation types including deposits, withdrawals, and transfers — with parallel consumers enabling concurrent workload handling. Designed to mirror M-Pesa-scale transaction volumes relevant to the Kenyan FinTech ecosystem.
Key Features
KRaft mode (no ZooKeeper dependency) · Atomic transactions with rollback support · Parallel consumer groups · Batch throughput benchmarking · PostgreSQL persistence with PgBouncer connection pooling · Redis caching layer for hot data
Tech Stack
Apache Kafka KRaft KafkaJS Node.js Express PostgreSQL PgBouncer Redis

pdfReplace
CLI Automation · Python · PyMuPDF
Overview
A command-line automation tool built for a water bottling company to streamline the generation of custom bottle label templates. Uses the PyMuPDF library to replace images embedded in a source PDF with a new design in a single command — eliminating the need for manual editing in design software for each label iteration and significantly reducing turnaround time for new label designs.
Key Features
Single-command image replacement in PDFs · PyMuPDF-powered PDF manipulation · CLI interface for easy use by non-developers · Significant reduction in label production time · Batch-compatible for multiple label variants
